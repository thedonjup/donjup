  <action>
    For each file, apply this mechanical transformation:

    1. Replace imports:
       - DELETE: `import { createServiceClient } from "@/lib/db/server"` or similar
       - DELETE: `import { createRentServiceClient } from "@/lib/db/rent-client"`
       - ADD: `import { db } from "@/lib/db"`
       - ADD: `import { aptTransactions, aptComplexes, ... } from "@/lib/db/schema"` (only tables used)
       - ADD: `import { eq, desc, asc, and, or, gte, lte, sql, inArray } from "drizzle-orm"` (only operators used)

    2. Replace query patterns:
       - `.from("apt_transactions").select("col1,col2").eq("field", val).order("field", { ascending: false }).limit(N)`
         becomes:
         `db.select({ col1: aptTransactions.col1, col2: aptTransactions.col2 }).from(aptTransactions).where(eq(aptTransactions.field, val)).orderBy(desc(aptTransactions.field)).limit(N)`

       - `.single()` at the end means the old code destructures `{ data }` and expects one row.
         Replace with: `const [row] = await db.select(...).from(...).where(...).limit(1)` then use `row`.

       - `.in("field", [values])` becomes `.where(inArray(table.field, values))`
       - `.gte("field", val)` becomes `gte(table.field, val)`
       - `.neq("field", val)` becomes `ne(table.field, val)` (import `ne` from drizzle-orm)
       - `.is("field", null)` becomes `isNull(table.field)` (import `isNull`)
       - `.not("field", "is", null)` becomes `isNotNull(table.field)`
       - `.ilike("field", "%val%")` becomes `ilike(table.field, "%val%")`

    3. Handle data shape:
       - Old code: `const { data, error } = await supabase.from(...)...`
       - New code: `const data = await db.select(...)...` (Drizzle returns array directly, no { data, error } wrapper)
       - Remove all `if (error)` checks related to QueryBuilder results
       - The old code has error handling like `if (error) { console.error(...); return ...; }` — replace with try/catch if needed, or remove if the page can just throw (Next.js error boundary catches it)

    4. CRITICAL — column name casing:
       - Drizzle schema uses camelCase (tradePrice, regionCode)
       - Existing code destructures snake_case (trade_price, region_code)
       - Use explicit select aliases to preserve snake_case: `.select({ trade_price: aptTransactions.tradePrice, region_code: aptTransactions.regionCode })`
       - OR if the page consumes data internally without passing to components expecting snake_case, you may use camelCase and update destructuring
       - CHECK each file: if data is passed to child components or used in JSX with snake_case field access, you MUST use snake_case aliases

    5. `src/app/apt/[region]/[slug]/page.tsx` special case:
       - Uses BOTH `createServiceClient()` (for apt_transactions, apt_complexes) AND `createRentServiceClient()` (for apt_rent_transactions)
       - Replace both with `db` — same Drizzle instance handles all tables

    6. `src/app/apt/sitemap.ts`:
       - Uses `.select("slug,region_code").from("apt_complexes")` — straightforward migration

    After migrating each file, run `npx tsc --noEmit` to catch type errors immediately.
