    1. Run `npm install drizzle-orm` and `npm install -D drizzle-kit`.

    2. Create `src/lib/db/index.ts`:
       - Import `drizzle` from `drizzle-orm/node-postgres`, `Pool` from `pg`, and `* as schema` from `./schema`
       - Create singleton Pool with EXACT same config as `client.ts`: `ssl: { rejectUnauthorized: false }`, max: 10, idleTimeoutMillis: 20_000, connectionTimeoutMillis: 10_000
       - Export `const db = drizzle({ client: getPool(), schema })`
       - The `getPool()` here is a LOCAL function (not exported) — different from the one in `client.ts`
       - Keep `client.ts` untouched — it will be deleted in plan 04

    3. Create schema files for ALL 13 tables. Use `src/types/db.ts` as the source of truth for column names and types. Use `pgTable` from `drizzle-orm/pg-core`.

       Column type mapping rules:
       - `string` fields that are IDs/codes/names/dates-stored-as-text → `text("column_name")`
       - `number` fields for integers (floor, trade_price, total_units) → `integer("column_name")`
       - `number` fields for decimals (size_sqm, rate_value, change_rate) → `numeric("column_name")`
       - `boolean` → `boolean("column_name")`
       - `Record<string, unknown>` / `unknown` (raw_data, top_drops, etc.) → `jsonb("column_name")`
       - `string[]` (storage_urls, hashtags) → `text("column_name").array()`
       - Timestamps (created_at, updated_at) → `timestamp("column_name").defaultNow()`

       Tables to create (13 total):
       - apt_transactions (from AptTransaction interface)
       - apt_rent_transactions (from research schema example)
       - finance_rates (from FinanceRate interface)
       - apt_complexes (from AptComplex interface)
       - daily_reports (from DailyReport interface)
       - page_views (from PageView interface — note `id` is `integer` not `text`)
       - content_queue (from ContentQueue interface)
       - push_subscriptions (from PushSubscription interface)
       - seeding_queue (from SeedingQueue interface)
       - reb_price_indices (check usage in `fetch-reb-index` cron)
       - homepage_cache (check usage in `refresh-cache` cron)
       - analytics_daily (check usage in `analytics` cron)
       - instagram_posts (check usage in `post-instagram` cron)

       For each schema file, export: the table constant, `$inferSelect` type, and `$inferInsert` type.

    4. Create `src/lib/db/schema/index.ts` that re-exports all table schemas.

    5. Verify: `npx tsc --noEmit` must pass with no errors on the new files.
