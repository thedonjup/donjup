## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, chip inner padding |
| sm | 8px | Between card meta items, chip gap in horizontal scroll row |
| md | 16px | Card horizontal padding (px-4), page horizontal padding |
| lg | 24px | Section vertical gap, card list spacing |
| xl | 32px | Page section breaks |
| 2xl | 48px | Hero section vertical padding (mobile: py-10 = 40px, nearest token) |
| 3xl | 64px | Desktop hero py-20 |

Exceptions:
- Chip horizontal padding: 12px (`px-3`) — multiple of 4, used for compact chip inner padding in area selector
- Mobile touch targets: minimum 44px height on all interactive elements (enforced via `globals.css` `@media (max-width: 640px)` rule — already in codebase)
- Card minimum height: 64px (`minHeight: 64` on mobile transaction cards — existing pattern to preserve)
- Rank badge: 28px × 28px fixed (existing `.rank-badge` class — do not change)
- Chart mobile height: 280px (from CONTEXT.md decision #4 — not a multiple of 4 but prescribed)

## Typography

**Scope: mobile-only phase (< 640px). Desktop sizes are out of scope and must not be introduced.**

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / Meta | 12px | 400 | 1.4 |
| Body / Card title | 14px | 400 / 600 | 1.6 (body) / 1.4 (card title) |
| Section heading | 24px (text-2xl) | 600 | 1.2 |

**Font family:** `--font-sans` (Pretendard Variable → system-ui fallback chain)

**Source:** `globals.css` mobile media query sets `body { font-size: 14px; line-height: 1.6 }` at max-width 640px. Heading size from `new-highs/page.tsx` (`text-2xl` at mobile). Card title from existing card pattern (`text-sm font-semibold`).

**Constraint:** Exactly 3 sizes (12px, 14px, 24px). Exactly 2 weights (400, 600). Do not introduce new type sizes or weights in this phase.

## Interaction Contracts

### Mobile Card — Transaction Item
- **Tap target:** min-height 64px, full-width `<Link>` block
- **Tap feedback:** `WebkitTapHighlightColor: transparent`; `card-hover:active` scale(0.98) from globals.css
- **Layout:** two-row layout. Row 1: rank badge (28px) + apt name (truncate) + price (right-aligned). Row 2: pl-7 offset + region · area · date as single text string
- **No swipe actions.** Tap navigates to apt detail page.

### Area Chip Selector
- **Layout:** single horizontal row, `overflow-x-auto`, `whitespace-nowrap`, `-mx-4 px-4` to extend to viewport edges
- **Chip size:** `px-3 py-2` (12px × 8px), `text-sm` (14px), `rounded-full`
- **Active state:** `bg-brand-500 text-white border-brand-500`; inactive: `t-card t-border t-text-secondary`
- **Scroll behavior:** momentum scroll (`-webkit-overflow-scrolling: touch`); no scrollbar visible on mobile
- **No wrapping.** If chips overflow, user scrolls horizontally.

### MobileNav Backdrop
- **z-index:** backdrop at z-40, drawer at z-50 (existing)
- **Backdrop:** `position: fixed`, `inset: 0`, `background: rgba(0,0,0,0.5)`, `touchAction: 'none'`, `overscrollBehavior: 'none'`
- **Dismiss:** tap backdrop closes drawer (existing behavior to preserve)
- **Scroll lock:** when drawer open, `document.body.style.overflow = 'hidden'` (add if not present)

### Chart (PriceHistoryChart)
- **Mobile breakpoint:** `< 640px` (Tailwind `sm:`)
- **Height:** 280px on mobile (fixed, not responsive)
- **Y-axis:** show maximum 4 ticks; format as `억` shorthand (e.g. "5억" not "5,000만")
- **Tooltip:** full price on tap (existing recharts default behavior)
- **No legend overflow** on mobile — truncate or hide if needed

### Table → Card Pattern
- **Breakpoint:** `sm:` (640px) — hide table on `< 640px`, show card list
- **Pattern:** `<div className="space-y-2 sm:hidden">` for cards, `<div className="hidden sm:block">` for table
- **Card padding:** `px-4 py-4`
- **Card border:** `rounded-xl border t-border t-card`

## Layout Constraints (Mobile-Specific)

- **Safe area:** Map bottom sheet must add `padding-bottom: env(safe-area-inset-bottom)` to handle iPhone home indicator
- **Horizontal padding:** `px-4` (16px) is the standard page padding — do not change for new mobile components
- **Max width:** `max-w-6xl mx-auto` wrapper — do not add new max-width constraints inside mobile cards
- **Breakpoint reference:** `sm:` = 640px in Tailwind defaults (this project uses default Tailwind v4 breakpoints; no custom tailwind.config found — confirm with `@media (min-width: 640px)`)
- **Table hide breakpoint:** use `sm:hidden` / `hidden sm:block` at 640px (matches existing new-highs pattern already in codebase)
