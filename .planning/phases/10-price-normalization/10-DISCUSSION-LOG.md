## 저층 기준선 정의

| Option | Description | Selected |
|--------|-------------|----------|
| 3층 이하 고정 (현재) | 단순하고 일관성 있음 | |
| 전체 층수 하위 1/3 | 네이버 방식, 단지 총층수 필요 | |
| 혼용 (3층 절대 + 1/3 비율 중 큰 값) | 리서치 권고안 | |
| 5층 이하 고정 | 부동산114 방식 | |
| 평균가 이상만 표시 | 사용자 제안 — 동일 면적 평균가 기준 | ✓ (초기) |

**User's choice:** 동일 면적 평균가 기준으로 평균가 이상만 표시 → 검증 후 **이중 필터(3층 제외 + 중위가 90% 미만 제거)**로 수정

**Notes:** 평균가 기반은 거래 건수 부족 시 통계적 무의미 + 시장 하락 은폐 위험으로 부적합. 중위가 90% 필터가 동일 효과를 더 안전하게 달성.

## 보정계수 / 필터링 방식

| Option | Description | Selected |
|--------|-------------|----------|
| 저층 제외 + 이상거래 필터 | 보정 없이 깨끗한 데이터만 | |
| 보정계수 적용 (1층-13%, 2층-10%) | 저층도 고층 환산해서 포함 | |
| 하이브리드 (제외 + 토글) | 기본 제외, 토글로 보정 포함 | |
| 하위 40% 제거 (사용자 제안) | 가격 순위 하위 40% 제거 | |
| 둘 다 적용 (3층 제외 + 중위가 90%) | 이중 필터 | ✓ |

**User's choice:** 3층 이하 물리적 제외 + 최근 6개월 중위가 90% 미만 통계적 제거 — 이중 필터

**Notes:** 사용자는 "비싼 가격이 보이는 게 좋다"고 함. 하위 40% 제거는 시장 하락 시 차트 업데이트 안 되는 문제. 중위가 90% 필터가 이상거래만 잡으면서 트렌드 보존.
