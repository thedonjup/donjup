Create `src/lib/price-normalization.ts` with the following exports:

1. **`LOW_FLOOR_MAX = 3`** — move constant here from AptDetailClient (AptDetailClient will import it)

2. **`isDealSuspicious(tradePrice: number, medianPrice: number, dealType: string | null): boolean`**
   - Returns true if `dealType === '직거래'` AND `tradePrice < medianPrice * 0.70`
   - Per NORM-04: 직거래 + 시세 대비 30% 이상 저가 = 이상거래

3. **`isDirectDeal(dealType: string | null): boolean`**
   - Returns true if `dealType === '직거래'`

4. **`computeMedianPrice(prices: number[]): number`**
   - Sort prices ascending, return middle value (or average of two middle for even count)
   - Return 0 if empty array

5. **`computeMovingMedian(data: { month: string; prices: number[] }[]): { month: string; median: number; count: number; isLowConfidence: boolean }[]`**
   - For each month, collect prices from current month + previous 2 months (3-month window)
   - Compute median of the collected prices
   - `isLowConfidence = true` when total collected prices < 5 (per CONTEXT: 5건 미만 = 점선)
   - Return sorted by month ascending

6. **`filterTransactions<T extends { floor: number; trade_price: number; deal_type: string | null }>(txns: T[], opts: { excludeLowFloor: boolean; recentMedian: number }): { normal: T[]; directDeals: T[]; excluded: T[] }`**
   - `normal`: transactions that pass all filters (not low floor if excludeLowFloor, not suspicious, not direct deal)
   - `directDeals`: transactions where `isDirectDeal(t.deal_type)` AND NOT suspicious (these render as gray dots)
   - `excluded`: suspicious transactions (직거래 + 중위가 90% 미만) — hidden entirely
   - Low floor logic: `t.floor <= LOW_FLOOR_MAX` when `excludeLowFloor` is true
   - Note: per CONTEXT decision, the filter threshold for exclusion is 90% of median. The `isDealSuspicious` function uses 70% for "이상거래", but the CONTEXT says "중위가의 90% 미만 거래도 제외" as additional filter. So `excluded` should include any transaction where `trade_price < recentMedian * 0.90` regardless of deal type.
   - User chose exclusion of low floors (not conversion/adjustment) per discuss-phase Decision 1.

7. **`groupByMonth(txns: { trade_date: string; trade_price: number }[]): { month: string; prices: number[] }[]`**
   - Group transactions by YYYY-MM from trade_date
   - Return sorted by month ascending

All functions are pure (no side effects, no DB calls). Types should use generics where possible so they work with both the full AptTransaction and the chart's simpler Transaction type.
