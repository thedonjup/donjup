# 한국 부동산 서비스 벤치마킹

**Domain:** 한국 아파트 실거래가 / 부동산 데이터 서비스
**Researched:** 2026-03-27
**Overall confidence:** MEDIUM — 주요 서비스 공개 정보 + 앱스토어/블로그 분석 기반. 실제 앱 내부 기능은 직접 접속 없이 간접 확인.

## 1. 서비스별 핵심 기능 정리

### 1.1 호갱노노 (hogangnono.com)

**포지셔닝:** 아파트 투자/거주 결정 올인원 플랫폼. "1등 실거래가 앱" 자처.

| 기능 | 상세 |
|------|------|
| 가격 차트 | 단지별 거래 변동 그래프 (시계열). 면적별 분리 여부 미확인. |
| 기간 선택 | 특정 기간 선택 UI 있음 (정확한 단위 미확인) |
| 저층 처리 | 명시적 필터링 없음. 거래 데이터에 층 정보 표시만 |
| 전세가율 | 필터 조건으로 전세가율 범위 설정 가능. 단지별 수치 표시 |
| 갭 가격 | 매매가 - 최고전세가 계산값을 지도 위에 표시하는 "갭투자" 필터 존재 |
| 신고가 랭킹 | 가장 많이 오른 아파트 랭킹. 기준: 직전거래 대비 변동률 추정 |
| 폭락 랭킹 | 가장 많이 내린 단지 표시 |
| 지수/인덱스 | 없음 (자체 지수 미제공) |
| 금리 | 없음 (금리 정보 별도 미제공) |
| 독자 기능 | **3D 일조량 분석** (시간/계절별), 학원 밀집도+시간당 비용, 외지인비율, 학군 (초중고 전부), 재건축 지표 (용적률/건폐율 필터), 주민 리뷰, 관심 단지 알림 |

**평가:** 데이터 레이어 가장 풍부. 분석 기능보다 검색/필터 UX에 강점.

### 1.2 아실 (asil.kr)

**포지셔닝:** 실거래가 데이터 전문. "동호수까지" 보여주는 디테일이 차별점.

| 기능 | 상세 |
|------|------|
| 가격 차트 | 2006년~현재 매매+전세 동시 비교 차트. 연간/기간 선택 가능. |
| 면적 분리 | 면적별 분리 차트 제공 여부 — 명시 확인 어려움 (MEDIUM 신뢰도) |
| 동호수 데이터 | **동·층·호수 단위 거래내역** 조회 (경쟁사 대비 가장 세밀) |
| 동별 보기 | 같은 단지 내 동(Building)별 가격 차이 비교 |
| 전세가율 | 자체 계산값 단지 카드에 표시 |
| 갭 투자 | 매매-전세 갭 금액 필터 제공 |
| 신고가 랭킹 | 최고가 실거래가 랭킹 + 일별 신고가 분석 (아파트미와 연계 또는 독자) |
| 아파트 비교 | A단지 vs B단지 차트 비교 |
| 가격지수 | 자체 Price Index 페이지 (asil.kr/asil/sub/price_index.jsp) — 상세 내용 접근 불가 |
| 미분양 | 지역별 미분양 현황 제공 |
| 학군 | 학업성취도 + 특목고 진학률로 학군 비교 |
| 이사 가구도 | 전출입 인구 데이터 시각화 |
| 독자 기능 | **일별 신고가 분석** (대한민국 최초 주장), 투자 고수 영상 통합 |

**평가:** 데이터 디테일(동호수, 비교차트) 최강. 투자자 타겟. UI는 정보 밀도 높아 복잡.

| 공시가격 | **동호수별 공시가격** 표시 (세금 계산 용도) |

| 일조량 | 시간/계절별 일조 분석 |

| 금리 | 없음 |
| 독자 기능 | 공시가격 조회, VR 매물, 네이버 생태계 연동 (지도/블로그/카페) |

**평가:** 매물 중개에 강점. 분석 기능은 경쟁사 대비 약함. 사용자 기반(트래픽) 최대.

### 1.4 KB부동산 (kbland.kr / data.kbland.kr)

**포지셔닝:** 금융기관 기반 시세 신뢰도 + 빅데이터 통계 플랫폼.

| 기능 | 상세 |
|------|------|
| 가격지수 | **KB 아파트 매매가격지수** — 주간/월간, 시도별/시군구별. 국가 공식 통계 수준 신뢰도 |
| 전세가율 | **지역별 전세가율 지도** (KB 통계기상도). 광역시/도 단위 |
| PIR | **소득대비 주택가격 배율(PIR)** 시계열 차트 제공 |
| 미분양 | 전국 미분양 아파트 현황 (시도별, 월별 추이) |
| 투자테이블 | 단지별 매매시세+전세시세+전세가율 통합 조회 테이블 |
| AI 예측시세 | KB AI 추정가 (KB 시세 기반 ML 모델 추정) |
| 통계 대시보드 | 매매가격지수, 전세가격지수, 매매/전세 변동률, 거래량 한 화면 요약 |
| 군집별 분석 | 없음 (강남3구 등 군집 인덱스 미제공 — MEDIUM 신뢰도, 데이터허브 고급 기능에는 있을 수 있음) |
| 독자 기능 | KB 시세 (금융권이 LTV 산정에 사용하는 공식 시세), PIR 시계열, 통계기상도 UI |

**평가:** 통계/지수 카테고리 최강. 투자자/전문가 타겟 데이터 플랫폼. 일반 소비자 UX는 불편.

## 3. 가격 차트 / 그래프 심층 분석

### 3.1 면적별 분리

- **아실**: 2006년~현재 매매+전세 동시 비교가 핵심. 면적별 탭 전환 제공 추정 (MEDIUM).
- **네이버 부동산**: 타입(면적)별 탭으로 차트 전환 가능 (MEDIUM).
- **호갱노노**: 단지 전체 평균 차트가 메인. 면적 분리는 필터로 접근.
- **돈줍 현황**: 면적별 차트 분리 구현됨 (`src/components/charts/PriceHistoryChart.tsx`). 경쟁우위.

### 3.2 기간 선택

- 대부분의 서비스: 1년/3년/5년/전체 탭 방식.
- **돈줍 현황**: 기간 선택 UI 미확인. 추가 여지 있음.

### 3.3 저층 거래 처리

- **모든 서비스**: 별도 처리 없이 전체 거래 포함. 저층 이상치는 사용자가 육안으로 파악.
- **돈줍 현황**: `docs/10-crash-criteria-v2.md`에 3단계 분류 체계 설계됨. 신고가/폭락 랭킹에서 이미 고층 위주 필터 적용. 이 점이 **차별점**.
- **기회**: 차트에서 저층 거래를 별도 색상으로 표시하거나 옵션으로 제외하는 기능은 경쟁사 중 없음.

### 3.4 차트 유형

- 업계 표준: **꺾은선 (Line chart)** — 실거래가 추이에 적합.
- 아실: 점(Scatter) + 선 혼합 방식 추정.
- 캔들스틱: 부동산 서비스에서 사용하는 곳 없음 (주식 데이터 아니라 거래 빈도 낮아 부적합).
- **돈줍 현황**: Recharts 기반 꺾은선 차트. 적합.

### 3.5 전세가율 차트

- **아실**: 매매+전세 동시 차트에서 전세가율 추이 계산 가능 (선택적 오버레이 여부 불명확).
- **KB**: 지역 단위 전세가율 시계열.
- **돈줍**: 없음 → **가장 임팩트 있는 추가 기능 후보 1순위**.

## 4. 지수 / 인덱스

### 업계 표준 지수

| 지수 | 출처 | 주기 | 공개 API |
|------|------|------|---------|
| KB 아파트 매매가격지수 | KB국민은행 | 주간/월간 | data.kbland.kr (무료) |
| 한국부동산원 아파트 매매가격지수 | 한국부동산원 | 주간/월간 | reb.or.kr Open API |
| 국토교통부 실거래가격지수 | 국토부 | 월간 | data.go.kr Open API |

### 서비스별 지수 제공

- **KB데이터허브**: 주력 기능. 지역별/전국 지수 차트, 등락률 히트맵.
- **아실**: 자체 Price Index 페이지 운영 중 (접근 실패, 상세 불명확).
- **부동산원 R-ONE**: 공식 통계 시스템. 차트 UI 있음.
- **호갱노노/직방/네이버**: 지수 미제공 (원데이터 표시만).

### 군집별 인덱스 (강남3구, 마용성, 노도강)

- 어떤 서비스도 명시적으로 **군집 인덱스**를 제공하지 않음.
- KB 투자테이블에서 구 단위 필터링은 가능하나, 군집 묶음 인덱스는 없음.
- **돈줍 기회**: 군집 인덱스는 블루오션. 강남3구/마용성/노도강 등을 묶은 자체 인덱스는 미디어 보도 가능한 독자 콘텐츠.

## 5. 랭킹 표현

### 5.1 신고가 기준 — 업계 관행

- **직전 거래 대비**: 가장 일반적. "직전 거래보다 OO만원 올랐다"
- **역대 최고가 대비**: 아파트미 스타일. 절대적 신고가.
- **최근 N개월 내 최고가 대비**: 호갱노노 추정 방식.

**돈줍 현황**: `docs/10-crash-criteria-v2.md` 참고 필요. 3단계 분류 체계로 단순 변동률이 아닌 복합 기준 적용 중. 이는 업계 내 차별화 요소.

### 5.2 저층 필터링 여부

- 경쟁사 전체: 랭킹에서 저층 필터링 없음.
- **돈줍**: 고층 필터 적용. 이상치 제거로 랭킹 신뢰도 향상. 차별점.
- 표시 방법: 필터 적용 여부를 사용자에게 명시하면 신뢰도 UX 향상.

### 5.3 변동률 단위

- 만원 단위 (절대값): 돈줍 현행
- % 단위 (상대값): 추세 비교에 유리
- 두 가지 병렬 표시: 아실 방식 추정

## 6. 금리 표현

### 현황 조사

- **모든 부동산 앱**: 금리 정보 비제공 (직방/호갱노노/아실/네이버/KB앱 모두).
- **뱅크샐러드/기준금리 전용 앱**: 금리 비교 특화.
- **부동산 계산기 서비스** (xn--989a00af8jnslv3dba.com 등): 금리 입력 기반 이자 계산.

### 돈줍 포지션

- 금리 대시보드 + 대출 이자 계산기를 부동산 앱에서 통합 제공 = **업계 내 유일 포지션**.
- 현재 표시 방식: 복수 금리 나열 추정.
- **개선 기회**: 대표 금리 1개 (기준금리 또는 주담대 평균금리)를 헤더/홈에 상시 노출하여 맥락 제공. 상세 페이지에서 전체 나열.

## 7. 돈줍에 추가할 기능 — 우선순위 분류

### 7.1 바로 적용 가능 (현재 DB 데이터 존재)

#### A. 전세가율 (Jeonse Rate)

**현황**: DB에 매매가(transactions)와 전세가(rents) 테이블 존재.
**구현**: 단지별 최근 전세가 / 최근 매매가 × 100.
**표시 위치**: 아파트 상세 → 현재 가격 섹션에 숫자 추가.
**임팩트**: 경쟁사 대부분이 제공하는 기본 지표. 부재 시 서비스 불완전 인상.
**데이터 이슈**: 매매·전세 거래 시점 불일치로 이상치 발생 가능 → 최근 3개월 내 각 1건 이상 있는 경우만 계산하고 없으면 "정보 부족" 표시.

```
전세가율 = (최근 전세 실거래가 ÷ 최근 매매 실거래가) × 100
```

#### B. 갭 금액

**현황**: 전세가율 계산 가능해지면 갭 금액은 자동 산출.
**구현**: 갭 = 최근 매매가 - 최근 전세가.
**표시 위치**: 아파트 상세 → 현재 가격 섹션에 추가.
**임팩트**: 투자자가 자주 찾는 지표.

#### C. 기간 선택 옵션 (차트)

**현황**: PriceHistoryChart가 있으나 기간 선택 UI 여부 불명확.
**구현**: 1년/3년/5년/전체 탭 → 기존 데이터 필터링.
**임팩트**: 경쟁사 모두 제공. 부재 시 UX 열위.

#### D. 전세가율 차트 (오버레이)

**현황**: 매매+전세 차트 이미 구현됨.
**구현**: 차트에 전세가율 추이 라인 오버레이 (2차 Y축).
**임팩트**: 아실도 제공하는 기능. 투자자가 갭 리스크 파악에 사용.

#### E. 신고가 기준 명시화

**현황**: 신고가 랭킹 있음.
**개선**: "역대 최고가 경신" vs "직전 거래 대비 최고" 기준을 UI에 명시.
**임팩트**: 사용자 신뢰도 향상.

---

### 7.2 데이터 추가 수집 필요

#### A. 지역 가격지수 (KB/부동산원 API 연동)

**데이터 출처**: 한국부동산원 Open API (무료, data.go.kr 신청)
**구현**: API 크론잡으로 주간/월간 지수 수집 → 트렌드 차트 페이지에 지역 지수 추가.
**임팩트**: KB데이터허브 외에 일반 소비자용 지수 차트 없음 → 차별화.
**비용**: API 무료, 크론잡 추가만.

#### B. 군집별 인덱스 (자체 산출)

**데이터 출처**: 기존 transactions DB + 군집 매핑 테이블.
**구현**: 강남3구(강남/서초/송파), 마용성(마포/용산/성동), 노도강(노원/도봉/강북), 비강남권 등 군집을 정의하고 자체 지수 산출.
**임팩트**: 어떤 서비스도 없음. 미디어/SNS 노출 가능성 높음.
**난이도**: 군집 정의는 쉬움. 인덱스 기준시점 설정 필요.

#### C. 입주물량/공급 예측

**데이터 출처**: 한국부동산원 또는 부동산114 API (일부 유료).
**구현**: 지역별 향후 12개월 입주 예정 물량 차트.
**임팩트**: 부동산지인이 이미 제공. 따라하기 수준이지만 필수 정보.

#### D. 미분양 현황

**데이터 출처**: 국토교통부 미분양 통계 (월간, 공개 데이터).
**구현**: 시도별 미분양 수 및 추이.
**임팩트**: KB데이터허브 및 아실이 제공. 기본 통계 완성도 향상.

---

### 7.3 장기 고려 사항

#### A. PIR (소득대비 집값)

**데이터 출처**: KB 데이터허브에서 연 1~2회 발표. 또는 통계청 가계소득 + 아파트 가격 자체 계산.
**임팩트**: 고관심 지표이나 업데이트 주기 낮음 (분기/연간).
**구현 난이도**: 중간. 데이터 수집 자동화 필요.

#### B. 학군 정보

**데이터 출처**: 학업성취도평가 공개 데이터 (교육부), 학교정보공시서비스 (schoolinfo.go.kr).
**임팩트**: 호갱노노가 이미 강점. 따라가기 어려운 영역.
**권고**: 직접 구현보다 외부 서비스 링크로 대체.

#### C. 교통 접근성 (지하철역 거리)

**데이터 출처**: 카카오맵 API (단지 → 최인근 지하철역 거리).
**임팩트**: 호갱노노/네이버가 지도 기반으로 제공.
**권고**: 카카오맵 이미 연동 중. 단지 상세에 "최인근 역 X분 거리" 추가는 낮은 난이도.

#### D. AI 가격예측

**임팩트**: 리치고가 주력 차별점으로 내세움.
**권고**: 충분한 거래량 데이터 없는 단지 많음. 예측 정확도 신뢰성 확보 어려움. 장기 고려.

#### E. 동호수별 가격 이력

**임팩트**: 아실의 최강 차별점.
**데이터**: 현재 DB에 층 정보 있음. 동/호수는 없음 (공공 데이터 한계).
**권고**: 공공 API가 동호수 제공 시 추가. 현재는 불가.

## 9. 돈줍 차별화 포인트 재확인

벤치마킹 결과, 돈줍이 **경쟁사 대비 유일하게 보유한 기능**:

1. **폭락/신고가 랭킹** — 모든 경쟁사가 없거나 약함. 돈줍의 핵심 가치.
2. **금리 대시보드 + 대출 계산기** — 부동산 앱에서 금리 정보 제공은 돈줍이 유일.
3. **3단계 폭락 분류 체계** — 단순 가격 하락이 아닌 복합 기준. 명시하면 신뢰도 강화.
4. **면적별 차트 분리** — 경쟁사 일부만 제공. 돈줍은 이미 구현됨.

**돈줍이 부재하는 업계 기본 기능**:

1. 전세가율 (단지별) — 가장 시급
2. 갭 금액 — 시급
3. 지역 가격지수 — 중요
4. 기간 선택 (차트) — 시급
