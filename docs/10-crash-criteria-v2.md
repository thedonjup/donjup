# 폭락 기준 리디자인 v2

## 1. 기존 기준 문제점

### 현행 로직
- **비교 기준**: 역대 최고가 (전체 거래 이력 중 최고가)
- **폭락 판정**: `change_rate <= -20%` → `is_significant_drop = true`
- **단계**: 1단계 (폭락 or 정상, 이진 분류)

### 문제점

| 문제 | 설명 | 영향 |
|------|------|------|
| 시간 왜곡 | 16년 전 최고가와 비교 시 현실감 없음 | 사용자 신뢰도 하락 |
| NULL 99.4% | 대부분의 데이터에 highest_price가 NULL | 폭락 탐지 자체가 불가 |
| 단일 단계 | -21%와 -50%를 동일하게 "폭락" 표시 | 심각도 구분 불가 |
| 과거 버블 왜곡 | 2021년 버블 최고가 기준 → 거의 모든 거래가 "폭락" | 정보 가치 없음 |

## 2. 시장조사 근거

### 금융시장 통용 기준
- **-10% (조정, Correction)**: 주식시장에서 통용되는 조정 구간
- **-20% (약세장, Bear Market)**: 공식적 약세장 진입 기준

### 한국 부동산 역사적 하락률
- **IMF 위기 (1997)**: 전국 평균 -12.4% (역대 최대)
- **글로벌 금융위기 (2008)**: 수도권 -5~8%
- **2022-23 금리 인상기**: 서울 일부 단지 -20~30%

### 언론 용어 사용 패턴
- **-10% 이상**: "약세", "하락세"
- **-15% 이상**: "폭락" 사용 시작
- **-25% 이상**: "대폭락", "급락" 등 강한 표현

### 경쟁사 차별화
- "폭락"을 메인 키워드로 쓰는 부동산 서비스 없음
- 돈줍만의 차별화 포인트로 활용 가능

## 3. 신규 기준 확정

### 비교 기준 변경
- **기본**: 최근 3년 최고가 (`trade_date >= 오늘 - 3년`)
- **Fallback**: 3년 내 거래 없으면 역대 최고가 사용

### 3단계 분류 체계

| drop_level | 범위 | 한국어 라벨 | 색상 | 의미 |
|-----------|------|----------|------|------|
| `normal` | 0% ~ -10% | - | - | 정상 변동 |
| `decline` | -10% ~ -15% | 하락 | 주황 (#f59e0b) | 주의 필요 |
| `crash` | -15% ~ -25% | 폭락 | 빨강 (#ef4444) | 적극 관심 |
| `severe` | -25% 이하 | 대폭락 | 진한 빨강 (#dc2626) | 극심한 하락 |

### is_significant_drop 기준
- 기존: `change_rate <= -20%`
- 변경: `change_rate <= -15%` (폭락 이상)

### 보조 지표
- 직전 거래 대비 변동률 (향후 구현 예정)

## 4. 사용자 관점 검증

### 투자자 김씨 (30대)
> "래미안 퍼스티지가 3년 전 25억이었는데 지금 19억? -24% 폭락이네. 매수 기회인가?"

- 3년 기준이라 현실감 있음
- 역대 최고가(2021년 30억) 대비 -37%보다 훨씬 신뢰감 있는 수치
- 하락/폭락/대폭락 3단계로 매수 타이밍 판단 용이

### 실수요자 이씨 (40대)
> "이 동네 평균은 안 떨어졌는데 이 단지만 -18% 폭락? 무슨 문제가 있나?"

- 단지별 비교가 직관적
- 하락/폭락 구분으로 심각도 파악 가능
- "하락" 단계가 있어 과도한 공포 유발 방지

### 임차인 박씨 (20대)
> "전세가율이 80%인데 매매가가 -15% 폭락? 깡통전세 위험?"

- 폭락 데이터 + 전세가율 연계가 임차인에게 실질적 가치
- 3단계 분류로 위험 수준 직관적 파악

## 5. UI 표시 방식

### 히어로 섹션 (page.tsx)
- `drop_level`에 따라 라벨 텍스트 변경: "하락" / "폭락" / "대폭락"
- 색상도 단계별 차별화

### 랭킹 탭 (RankingTabs.tsx)
- 각 항목 우측에 drop_level 배지 표시
- decline: 주황 배경 + "하락"
- crash: 빨강 배경 + "폭락"
- severe: 진한 빨강 배경 + "대폭락"

### 거래 이력 테이블 (TransactionTabs.tsx)
- 변동률 셀에 drop_level 배지 추가 표시
- 기존 변동률 표시 옆에 라벨 추가

## 6. 코드 변경 명세

### DB (supabase/schema.sql)
```sql
ALTER TABLE apt_transactions ADD COLUMN IF NOT EXISTS drop_level VARCHAR(10) DEFAULT 'normal';
-- 'normal' | 'decline' | 'crash' | 'severe'
```

### API - fetch-transactions/route.ts
- `enrichTransactions()`: 최고가 조회 시 최근 3년 필터 추가
- 3년 내 거래 없으면 역대 최고가 fallback
- `drop_level` 계산 로직 추가
- `is_significant_drop` 기준 -20% → -15%

### API - validate-data/route.ts
- 재계산 시 `drop_level` 함께 계산
- `is_significant_drop` 기준 -20% → -15%
- 폭락 플래그 불일치 검증도 -15% 기준으로 변경

### UI Components
- `page.tsx`: 히어로 텍스트 drop_level 기반 동적 변경
- `RankingTabs.tsx`: Transaction 인터페이스에 `drop_level` 추가, 배지 렌더링
- `TransactionTabs.tsx`: Transaction 인터페이스에 `drop_level` 추가, 배지 렌더링
