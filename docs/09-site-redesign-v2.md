## Phase 1: 기획

### 1-1. 핵심 컨셉 재정의

**기존 포지셔닝:**
> "전국 아파트 실거래가 폭락/신고가 랭킹과 금리 정보"

**새 포지셔닝:**
> "전국 아파트 실거래가 폭락/신고가 추적 + 투자 분석 플랫폼"
> — 자극적인 데이터 큐레이션(공포/FOMO) + 실용적인 분석 도구

**경쟁사 대비 빈틈 공략:**

| 빈틈 | 경쟁사 현황 | 돈줍 전략 |
|------|-----------|----------|
| 극단값 중심 정보 | 호갱노노/아실은 "평균 시세" 중심 | 폭락/신고가 TOP 일일 자동 랭킹 |
| 금리 체감형 | KB부동산은 시세 기준, 금리→이자 변화 실시간 체감 없음 | 금리 변동 → 내 이자 변화 시뮬레이터 |
| 감정 자극형 콘텐츠 | 기존 플랫폼은 "정보 나열"에 집중 | "O아파트 최고가 대비 -45% 폭락" 같은 헤드라인 |
| 일일 자동 업데이트 | 대부분 주간/월간 단위 | 매일 아침 자동 생성 랭킹 |
| 전월세 극단값 | 전세 급등/급락 특화 서비스 없음 | 전세 급등 TOP, 전세가율 위험 단지 알림 |

### 1-2. 새 페이지 구조 (Sitemap)

```

```

### 1-4. DB 테이블 현황 및 신규 필요 사항

**현재 보유 테이블:**
- `apt_complexes` — 단지 마스터 (region_code, apt_name, slug, total_units, built_year 등)
- `apt_transactions` — 매매 실거래 (trade_price, change_rate, is_new_high, is_significant_drop, deal_type)
- `apt_rent_transactions` — 전월세 실거래 (deposit, monthly_rent, rent_type, contract_type)
- `finance_rates` — 금리 지표 (rate_type, rate_value, change_bp)
- `daily_reports` — 데일리 리포트 (top_drops, top_highs, rate_summary)
- `reb_price_indices` — 한국부동산원 가격지수 (index_type, index_value)
- `page_views` — 조회수

**신규 필요:**
- 검색 인덱스: `apt_complexes`에 Full-Text Search 인덱스 추가 (또는 Supabase의 `ilike` 활용)
- 미분양 데이터: `unsold_housing` 테이블 (region_name, unsold_count, base_date) — 트렌드 페이지용
- 전세가율 계산 뷰: `v_jeonse_ratio` — 단지별 최근 매매가 대비 전세가율 자동 계산

### 2-3. 단지 상세 페이지 (/apt/[region]/[slug]) 재설계

**현재 문제점:**
- 면적이 ㎡ 단위만 표시 → 평형 미표기
- 매매/전월세가 같은 열에 나열 → 구분 어려움
- 전세가율이 단지 수준에서만 표시 → 면적별 전세가율 없음
- 사이드바에 대출 계산기 연결이 약함 → 현재 거래가 기반 자동 계산 없음

   CREATE OR REPLACE VIEW v_jeonse_ratio AS
   WITH latest_trade AS (
     SELECT DISTINCT ON (apt_name, region_code)
       apt_name, region_code, region_name, trade_price, size_sqm
     FROM apt_transactions
     ORDER BY apt_name, region_code, trade_date DESC
   ),
   latest_jeonse AS (
     SELECT DISTINCT ON (apt_name, region_code)
       apt_name, region_code, deposit, size_sqm
     FROM apt_rent_transactions
     WHERE rent_type = '전세'
     ORDER BY apt_name, region_code, trade_date DESC
   )
   SELECT
     t.apt_name, t.region_code, t.region_name, t.size_sqm,
     t.trade_price, j.deposit,
     ROUND((j.deposit::numeric / t.trade_price::numeric) * 100, 1) AS jeonse_ratio
   FROM latest_trade t
   JOIN latest_jeonse j ON t.apt_name = j.apt_name AND t.region_code = j.region_code
   WHERE t.trade_price > 0
   ORDER BY jeonse_ratio DESC;

## 부록: 기술 참고 사항

### A. 면적 변환 공식

```typescript
// 1평 = 3.305785㎡ (정확값)
// 관행적으로 3.306으로 반올림

export function sqmToPyeong(sqm: number): number {
  return Math.round(sqm / 3.306);
}

export function formatArea(sqm: number): string {
  return `${sqm}㎡(${sqmToPyeong(sqm)}평)`;
}

// 사용 예:
// formatArea(84.99) → "84.99㎡(26평)"
// formatArea(59.97) → "59.97㎡(18평)"
// formatArea(114.8) → "114.8㎡(35평)"
```

### B. 검색 성능 최적화

```sql
-- pg_trgm 확장으로 유사 문자열 검색 (오타 허용)
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE INDEX idx_complexes_name_trgm ON apt_complexes
  USING gin (apt_name gin_trgm_ops);

-- 검색 쿼리 예시 (오타 허용 포함)
SELECT *, similarity(apt_name, '래미안') AS sim
FROM apt_complexes
WHERE apt_name % '래미안' OR apt_name ILIKE '%래미안%'
ORDER BY sim DESC
LIMIT 20;

-- 대안: Supabase Full-Text Search
ALTER TABLE apt_complexes ADD COLUMN fts tsvector
  GENERATED ALWAYS AS (to_tsvector('simple', apt_name || ' ' || region_name || ' ' || COALESCE(dong_name, ''))) STORED;
CREATE INDEX idx_complexes_fts ON apt_complexes USING gin(fts);
```

### C. 전세가율 위험도 기준

| 전세가율 | 등급 | 표시 | 의미 |
