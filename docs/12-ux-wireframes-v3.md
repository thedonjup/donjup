## 1. 설계 원칙

### 1-1. 핵심 원칙

| 원칙 | 설명 | 적용 |
|------|------|------|
| 정보 밀도 높되 읽기 쉽게 | 한 화면에 많은 데이터를 보여주되, 시각적 계층 구조로 인지 부하 최소화 | 카드 내 정보 3단계 구분 (제목/핵심수치/부가정보) |
| 자극적이되 과하지 않게 | 빨강(폭락)/초록(상승) 시멘틱 컬러는 수치 옆에만 제한 사용 | 배경은 중성 톤, 컬러는 데이터 포인트에만 |
| 모바일 퍼스트 | 터치 타겟 44px 이상, 한 손 사용 최적화 | 하단 네비바, 스와이프 제스처 |
| 다크모드 완전 지원 | 기존 CSS 변수 시스템 유지 확장 | 신규 컴포넌트 모두 `t-*` 클래스 사용 |
| 점진적 공개 | 처음에는 핵심 정보만, 상세는 탭/토글로 | 단지 상세 6탭 구조 |

### 1-2. 타이포그래피 계층

```
H1: text-4xl font-black (히어로 헤드라인)
H2: text-2xl font-extrabold (섹션 제목)
H3: text-lg font-bold (카드 제목)
본문: text-sm (14px) — 기본 텍스트
캡션: text-xs (12px) — 부가 정보
마이크로: text-[11px] — 배지, 변동률
숫자: tabular-nums font-bold — 모든 금액/수치
```

### 1-3. 컬러 시멘틱 확장

```
기존 유지:
  --color-semantic-drop: #ef4444 (폭락/하락)
  --color-semantic-rise: #10b981 (상승/신고가)

신규 추가:

  --color-map-high: #10b981 (신고가)
  --color-map-normal: #94a3b8 (일반)
  --color-category-apt: #3b82f6 (아파트)
  --color-category-villa: #8b5cf6 (빌라)
  --color-category-officetel: #ec4899 (오피스텔)

### 3-2. 컴포넌트 계층

```

  ├── <Logo />                        ← Link to "/"
  ├── <CategoryTabs />                ← 클라이언트 컴포넌트
  │     ├── 탭: 아파트 (기본 active)
  │     ├── 탭: 빌라

  │     └── 탭: 토지
  ├── <GlobalSearchBar />             ← 클라이언트 컴포넌트
  │     ├── 입력 필드 (placeholder: "단지명·지역 검색")
  │     └── <SearchDropdown />        ← 자동완성 결과
  ├── <AuthButton />                  ← 클라이언트 컴포넌트
  │     ├── 비로그인: "로그인" 텍스트 버튼
  │     └── 로그인: 프로필 아바타 + 드롭다운
  ├── <NotificationBell />            ← 클라이언트 컴포넌트
  │     ├── 벨 아이콘
  │     ├── 미읽음 배지 (빨간 점)
  │     └── <NotificationDropdown />  ← 알림 목록 패널
  └── <ThemeToggle />                 ← 기존 유지
```

### 3-3. 카테고리 탭 동작

```
상태 관리:
  - URL 기반: 쿼리파라미터 ?type=apt (기본값)
  - 전역 상태: Zustand 또는 URL searchParams
  - 모든 페이지에서 카테고리 필터 적용

탭 전환 시:
  1. URL 쿼리파라미터 변경 (?type=villa)
  2. 페이지 데이터 리페치 (서버 컴포넌트 재렌더)
  3. 활성 탭 하단 인디케이터 애니메이션 (2px solid brand-600)

아파트 탭 선택 시:
  - 드롭다운 표시: [전체 | 서울 | 경기 | 인천 | ...]
  - 지역 필터 적용

비아파트 탭:
  - 아직 데이터 없는 카테고리는 "준비 중" 배지 + 비활성

스타일:
  활성 탭: font-bold, border-bottom 2px brand-600, t-text
  비활성 탭: font-medium, t-text-tertiary, hover:t-text-secondary
```

### 3-4. 검색바 동작

```
기본 상태:
  [🔍 단지명·지역 검색...]  ← 축소 상태 (데스크톱 240px)

포커스 상태:
  ┌──────────────────────────────────────────┐
  │ 🔍 │래미안                        │ ✕    │
  ├──────────────────────────────────────────┤
  │ 🏠 래미안 퍼스티지 · 서초구 반포동        │
  │    최근 17.6억 | ▼45.0%                 │
  ├──────────────────────────────────────────┤
  │ 🏠 래미안 블레스티지 · 서초구 서초동      │
  │    최근 22.3억 | ▼12.5%                 │
  ├──────────────────────────────────────────┤
  │ 📍 "래미안" 전체 검색 결과 42건 →         │
  └──────────────────────────────────────────┘

동작:
  - 입력 2글자 이상 → 300ms 디바운스 → API 호출
  - API: GET /api/search/autocomplete?q=래미안&type=apt
  - 결과 최대 5건 + "전체 검색 결과 N건" 링크
  - Enter 키 → /search?q=래미안&type=apt 이동
  - 항목 클릭 → 해당 단지 상세 페이지 이동
  - ESC → 드롭다운 닫기
  - 모바일: 전체화면 검색 모드 전환
```

```

  │    28.5억 | 어제                  │
  ├──────────────────────────────────┤
  │ 💰 기준금리 인하 (3.25% → 3.00%) │
  │    3일 전                        │
  ├──────────────────────────────────┤
  │         알림 설정 관리 →           │
  └──────────────────────────────────┘

알림 유형:
  1. 관심 단지 가격 변동 (매매/전세)

  4. 시스템 공지

데이터 소스:
  - Supabase Realtime 또는 폴링 (30초 간격)
  - 테이블: notifications (user_id, type, payload, is_read, created_at)

## 11. 인터랙션 명세

### 11-1. 애니메이션/트랜지션

```
[공통 트랜지션]
  - 카드 호버: transform translateY(-2px) + shadow 변경, 200ms ease
  - 탭 전환: opacity fade 150ms
  - 모달 열기: opacity 0→1 200ms + scale 0.95→1 200ms
  - 모달 닫기: opacity 1→0 150ms + scale 1→0.95 150ms
  - 드롭다운: max-height 0→auto 200ms + opacity

[히어로 특수 효과]
  - live-dot: pulse 애니메이션 2초 주기 (기존)
  - 배지 색상: drop_level에 따라 동적 변경

[지도 마커]
  - 등장: scale 0→1 bounce 300ms
  - 선택: scale 1→1.2 200ms + ring 효과
  - 팝업: fade-in 200ms

[바텀시트 (모바일)]
  - 드래그: transform translateY, 터치 추적
  - 스냅: 3단계 높이로 스냅, 300ms spring
```

### 11-2. 키보드/접근성

```
[키보드 네비게이션]
  - Tab: 포커스 순서 (로고 → 카테고리 → 검색 → 로그인 → 알림)
  - Enter/Space: 버튼/링크 활성화
  - Escape: 모달/드롭다운 닫기
  - Arrow keys: 탭 전환, 드롭다운 항목 이동

[ARIA 속성]
  - 탭: role="tablist", role="tab", aria-selected
  - 모달: role="dialog", aria-modal="true", aria-labelledby
  - 알림 배지: aria-label="읽지 않은 알림 3건"
  - 차트: aria-label="가격 추이 차트"
  - 마커: aria-label="래미안퍼스티지 17.6억 폭락 45%"

[스크린 리더]
  - 가격 변동: "하락 45퍼센트" (시각적 ▼ 대신)
  - drop_level 배지: 텍스트로 읽힘 ("대폭락", "폭락", "하락")

    회색 배경 + 중앙 스피너

  댓글:
    입력 영역 스켈레톤 + 댓글 카드 스켈레톤 ×3

[에러 상태]
  - DB 연결 실패: "데이터를 불러올 수 없습니다. 잠시 후 다시 시도해 주세요."
  - 빈 데이터: "데이터 수집 중입니다. 매일 밤 자동으로 업데이트됩니다." (기존)
  - 네트워크 오류: 토스트 알림 + 재시도 버튼
```

### 11-4. 반응형 브레이크포인트

```
기존 Tailwind 기본값 사용:
  sm: 640px
  md: 768px   ← 모바일/태블릿 경계 (하단 네비바 표시 기준)
  lg: 1024px  ← 사이드바 표시 기준
  xl: 1280px

주요 반응형 변경점:
  < 768px (모바일):
    - 하단 네비바 표시
    - 사이드바 숨김
    - 테이블 → 카드 전환
    - 히어로 축소 (text-2xl)
    - 지도 바텀시트
    - 6탭 수평 스크롤

  768px ~ 1024px (태블릿):
    - 하단 네비바 숨김
    - 사이드바 숨김 (본문 전체 너비)
    - 히어로 중간 크기 (text-3xl)

  > 1024px (데스크톱):
    - 풀 네비게이션
    - 2/3 + 1/3 레이아웃
    - 히어로 최대 크기 (text-5xl)
    - 지도 좌측 패널
```

## 부록: 신규 DB 테이블

```sql
-- 사용자 프로필 (Supabase Auth 확장)
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  nickname TEXT,
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 관심 단지
CREATE TABLE favorites (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  slug TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, slug)
);

-- 알림 구독
CREATE TABLE notification_subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  slug TEXT,
  types TEXT[] DEFAULT '{price_change,new_trade}',
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 알림 로그
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  type TEXT NOT NULL,
  title TEXT NOT NULL,
  body TEXT,
  payload JSONB,
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 댓글
CREATE TABLE comments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  slug TEXT NOT NULL,
  content TEXT NOT NULL,
  likes INTEGER DEFAULT 0,
  parent_id UUID REFERENCES comments(id),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 댓글 좋아요 (중복 방지)
CREATE TABLE comment_likes (
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  comment_id UUID REFERENCES comments(id) ON DELETE CASCADE,
  PRIMARY KEY (user_id, comment_id)
);
```
